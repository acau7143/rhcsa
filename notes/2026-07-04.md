# Day 3 — SELinux / firewalld / journald

## 1-3. SELinux Port (httpd 비표준 포트)

### Question
A non-standard port 82 is used to run a web server that is having trouble providing service. Troubleshoot and fix the following conditions:
- Web server on the system must be able to serve all existing HTML files located in /var/www/html (note: do not remove or modify content of existing files)
- Web server must be configured to serve on port 82
- Web server must start automatically at system startup

### My Solution
```bash
# 1. 컨텍스트 확인 (문제 진단)
ls -Z /var/www/html/

# 2. 컨텍스트 교정 (혹시 몰라서, 안전장치)
semanage fcontext -a -t httpd_sys_content_t "/var/www/html(/.*)?"
restorecon -Rv /var/www/html

# 3. 포트 정책
semanage port -a -t http_port_t -p tcp {포트번호}

# 4. 서비스 + 방화벽
systemctl enable --now httpd
firewall-cmd --permanent --add-port={포트번호}/tcp
firewall-cmd --reload
```

### Verification
```bash
$ curl http://localhost:82
<!doctype html> ... HTTP Server Test Page powered by: Rocky Linux ...

$ firewall-cmd --list-ports
82/tcp

$ systemctl is-enabled httpd
enabled
```

### 내가 헷갈렸던 것 / 배운 것
- SELinux는 프로세스가 "원래 정해진 일"만 하도록 제한하는 보안 시스템. httpd는 기본적으로 80번 포트만 쓰도록 정해져 있어서, 82번을 쓰려면 SELinux한테 "이것도 허용해줘"라고 알려줘야 함.
- `semanage port -a -t http_port_t -p tcp 82` 한 줄로 82번 포트를 httpd가 써도 되는 목록에 추가.
- `setenforce 0`으로 SELinux를 끄고 작업하면 안 됨 — 진짜 설정이 맞는지 확인이 안 되기 때문에, 켠 상태로 검증까지 마쳐야 함.
- 이 문제는 3개 독립된 층위(httpd Listen 설정 / SELinux 포트 정책 / firewalld)를 모두 통과해야 함 — 하나만 빠지면 반쪽만 해결됨.

## 2. SELinux Boolean 설정 (httpd_enable_homedirs)

### Question
(공식 시험 범위 항목 — "Use boolean settings to modify system SELinux settings". 별도 덤프 문항 원문 없음, RHEL 10 공식 Exam Objectives 기반 자체 실습)

### My Solution
```bash
getsebool httpd_enable_homedirs
setsebool -P httpd_enable_homedirs on
```

### Verification
```bash
$ getsebool httpd_enable_homedirs
httpd_enable_homedirs --> on
```

### 내가 헷갈렸던 것 / 배운 것
- Boolean과 포트 정책(`semanage port`)은 완전히 다른 층위. 포트 정책은 "이 포트를 써도 되는가", boolean은 "이 서비스가 이 행동을 해도 되는가"를 결정.
- `-P` 옵션 필수 — 빠뜨리면 재부팅 시 초기화되어 시험에서 감점 포인트가 됨.

## 3. firewalld 단독 설정 (zone, port, service)

### Question
(daankwak 1-3 문항 안에 포함된 범위. 별도 원문 없음, firewalld 기본 조작 자체 실습)

### My Solution
```bash
firewall-cmd --get-active-zones
firewall-cmd --permanent --add-service=http
firewall-cmd --list-services   # reload 전
firewall-cmd --reload
firewall-cmd --list-services   # reload 후
```

### Verification
```bash
$ firewall-cmd --get-active-zones
public
  interfaces: enp0s3

$ firewall-cmd --list-services   # reload 전
cockpit dhcpv6-client ssh

$ firewall-cmd --reload
success

$ firewall-cmd --list-services   # reload 후
cockpit dhcpv6-client http ssh
```

### 내가 헷갈렸던 것 / 배운 것
- firewalld는 runtime(메모리, 즉시 적용)과 permanent(디스크 저장, 재부팅 시 반영) 두 층으로 나뉨.
  - `--add-port=X` (permanent 없이) → 지금 당장 접속 가능하지만 재부팅하면 사라짐
  - `--permanent --add-port=X` (reload 없이) → 지금은 적용 안 되지만 재부팅하면 적용됨
  - 둘 다 세트로 써야 "지금도 되고 계속 유지"되는 정상 상태
- `--reload` 전/후 `--list-services` 비교로 이 개념을 실제로 눈으로 확인함 — reload 전엔 http 안 보이다가 reload 후 나타남.
- 표준 서비스(http, https, ssh)는 `--add-service`, 비표준 포트(82, 8080 등)는 `--add-port`로 다르게 접근.

## 4. journald 영구 보존 설정

### Question
(공식 시험 범위 항목 — "Configure system logging" persistent storage. 별도 덤프 문항 원문 없음)

### My Solution
```bash
sed -i 's/^#Storage=auto/Storage=persistent/' /etc/systemd/journald.conf
mkdir -p /var/log/journal
systemctl restart systemd-journald
journalctl --flush
```

### Verification
```bash
$ grep ^Storage /etc/systemd/journald.conf
Storage=persistent

$ ls -la /var/log/journal/
drwxr-xr-x. 3 root root   46 Jul  4 07:32 .
drwxr-xr-x. 2 root root   28 Jul  4 07:32 97de4042572d4835890a1206ca3a8e97
```

### 내가 헷갈렸던 것 / 배운 것
- 기본 설정(`Storage=auto`)은 `/var/log/journal` 디렉토리가 없으면 로그를 메모리(`/run/log/journal`)에만 저장 → 재부팅 시 로그 소실. 실무에서 장애 원인 분석 시 로그가 사라지면 안 되므로 `persistent`로 바꾸는 게 핵심.
- `systemctl restart`만으로는 기존에 메모리에 있던 로그가 디스크로 안 옮겨질 수 있어서 `journalctl --flush`로 강제 반영 필요. (단, 시험 채점 기준에서는 설정 파일 + 재시작까지만 확인할 가능성이 높아 flush는 선택적으로 이해)
- `/var/log/journal/` 안에 생기는 폴더명은 machine-id 기반 — 그 안에 실제 `.journal` 로그 파일이 저장됨.

## 원래 상태로 되돌리는 명령어 (Rollback)

```bash
# 1. SELinux 포트 정책 제거
semanage port -d -t http_port_t -p tcp 82

# 2. httpd.conf에서 Listen 82 제거 (vi로 수동 삭제)
vi /etc/httpd/conf/httpd.conf

# 3. SELinux boolean 원복
setsebool -P httpd_enable_homedirs off

# 4. firewalld 서비스 제거
firewall-cmd --permanent --remove-service=http
firewall-cmd --reload

# 5. httpd 비활성화 (필요시)
systemctl disable --now httpd

# 6. journald 설정 원복
sed -i 's/^Storage=persistent/#Storage=auto/' /etc/systemd/journald.conf
systemctl restart systemd-journald
```
