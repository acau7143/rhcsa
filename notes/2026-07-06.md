## 9. find (소유자 조건으로 파일 찾기)

### Question
Find all files and directories owned by user `jacques` on this system and copy them into the `/root/findfiles` directory.

### My Solution
```bash
useradd jacques
su - jacques -c "touch /tmp/jacques_file1.txt"
su - jacques -c "touch /tmp/jacques_file2.txt"
su - jacques -c "mkdir /tmp/jacques_dir"

mkdir -p /root/findfiles
find / -user jacques -exec cp -a {} /root/findfiles \; 2>/dev/null
```

### Verification
```bash
$ ls -l /root/findfiles
-rw-rw----. 1 jacques mail    0 Jul  1 23:38 harry
-rw-rw----. 1 jacques mail    0 Jul  6 02:06 jacques
drwxr-xr-x. 2 jacques jacques 6 Jul  6 02:07 jacques_dir
-rw-r--r--. 1 jacques jacques 0 Jul  6 02:06 jacques_file1.txt
-rw-r--r--. 1 jacques jacques 0 Jul  6 02:06 jacques_file2.txt
```
jacques 소유 파일/디렉토리(홈 디렉토리, 메일함 포함)가 모두 정확히 복사됨. `cp -a`로 타임스탬프까지 원본 그대로 보존된 것 확인.

### 내가 헷갈렸던 것 / 배운 것
- `{}`는 find가 찾은 결과가 들어가는 자리, `\;`는 `-exec` 명령 종료 신호(`;`를 bash가 먼저 해석 못하게 이스케이프)
- `cp -a`를 안 쓰면 복사 시 소유자가 root로 바뀜 → 문제 의도가 깨짐
- **결과 저장 디렉토리(`/root/findfiles`)가 탐색 범위(`/`) 안에 있으면 순환 복사 문제 발생** (find가 자기가 복사한 결과물을 또 찾아서 자기 자신에게 복사하려다 `are the same file` 에러). 재실행 시 `rm -rf /root/findfiles && mkdir -p`로 초기화 필요
- `/root/findfiles`를 미리 `mkdir -p`로 안 만들면 `cp`가 실패함

## 10. grep (문자열 검색 + 순서 보존)

### Question
Search the file `/etc/passwd` for lines containing the string `bash` and save (in the same order) the matching lines to a new file `/root/bashusers`.

### My Solution
```bash
grep "bash" /etc/passwd > /root/bashusers
```

### Verification
```bash
$ cat /root/bashusers
root:x:0:0:root:/root:/bin/bash
reallinux:x:1000:1000::/home/reallinux:/bin/bash
testweb:x:1001:1001::/home/testweb:/bin/bash
jacques:x:1002:1002::/home/jacques:/bin/bash
```
UID 순서(0→1000→1001→1002)와 결과 파일 줄 순서가 일치 → grep이 원본 순서 그대로 보존했음을 확인.

### 내가 헷갈렸던 것 / 배운 것
- grep은 파일을 위→아래로 읽으며 매치되는 줄을 그 순서 그대로 출력하므로, 별도 정렬 없이도 "같은 순서" 요구사항이 자동으로 충족됨
- `>`(덮어쓰기) vs `>>`(추가) 구분 — 새 파일 생성이므로 `>`가 맞음
- `grep -v`는 반대(제외) 검색, `-i`는 대소문자 무시 — 문제를 잘 읽고 포함/제외 구분 필요

## 11. tar (bzip2 압축)

### Question
Create a compressed tar archive of the `/usr/local` directory using bzip2 compression, and save it as `/root/backup.tar.bz2`.

### My Solution
```bash
tar -cvjf /root/backup.tar.bz2 /usr/local
```

### Verification
```bash
$ file /root/backup.tar.bz2
/root/backup.tar.bz2: bzip2 compressed data, block size = 900k
```
매직 넘버 기준으로 실제 bzip2 압축 확인 완료 (확장자만이 아니라 내용 자체가 bzip2인지 검증).

### 내가 헷갈렸던 것 / 배운 것
- `tar` 자체는 압축이 아니라 "묶기"만 함. `-j`(bzip2)/`-z`(gzip)/`-J`(xz)는 tar가 압축 프로그램을 호출해서 파이프로 연결해주는 것
- `-f` 옵션은 항상 맨 뒤에 두고, 바로 다음 인자가 파일명이어야 함 (`-cvjf 파일명 대상` 순서 고정)
- `tar -tvf`는 압축 방식과 무관하게 목록을 보여주므로, 진짜 bzip2인지 확인하려면 `file` 명령어로 별도 검증 필요

## 12. ACL (setfacl/getfacl)

### Question
Copy `/etc/fstab` to `/root/fstab.orig`. On this copy:
- `natasha` user: read/write
- `harry` user: no access
- `admins` group: read only
- other: read only

*(실습 시 계정명 natasha/harry → testweb/jacques로 대체, admins 그룹 신규 생성)*

### My Solution
```bash
cp /etc/fstab /var/tmp/fstab
chown root:root /var/tmp/fstab
chmod 644 /var/tmp/fstab

groupadd admins
setfacl -m u:testweb:rw /var/tmp/fstab
setfacl -m u:jacques:--- /var/tmp/fstab
setfacl -m g:admins:r /var/tmp/fstab
```

### Verification
```bash
$ getfacl /var/tmp/fstab
# file: var/tmp/fstab
# owner: root
# group: root
user::rw-
user:testweb:rw-
user:jacques:---
group::r--
group:admins:r--
mask::rw-
other::r--
```
모든 요구사항(개별 사용자 rw/차단, 그룹 r, mask가 rw까지 커버) 충족 확인.

### 내가 헷갈렸던 것 / 배운 것
- `setfacl -m g:그룹명:권한`을 쓰려면 그 그룹이 **시스템에 실제로 존재**해야 함 → 없으면 `groupadd`로 먼저 생성
- `mask`는 named user/group ACL 권한의 **상한선**. mask가 낮으면 아무리 rw를 줘도 실제로는 mask 수준까지만 적용됨 → 반드시 `getfacl`로 최종 확인
- `ls -l`만으로는 ACL 적용 여부를 알 수 없음, 권한 뒤 `+` 표시로만 힌트를 얻고 `getfacl`로 세부 확인해야 함
- 개별 사용자 ACL이 그룹 권한보다 우선 적용됨

## 보충 1. Sticky Bit (1777)

### Question
(덤프 문항 아님 — Day 4 보충 학습, `/shared` 공용 디렉토리에서 sticky bit 동작 검증)

### My Solution
```bash
mkdir /shared
chmod 1777 /shared
```

### Verification
```bash
$ ls -ld /shared
drwxrwxrwt. 2 root root 6 Jul  6 04:00 /shared

# testweb이 파일 생성
$ su - testweb -c "touch /shared/test.txt"

# jacques가 삭제 시도 → 차단됨
$ su - jacques -c "rm /shared/test.txt"
rm: cannot remove '/shared/test.txt': Operation not permitted

# root는 삭제 가능 (예외 확인)
$ rm -rf /shared/
```

### 내가 헷갈렸던 것 / 배운 것
- 파일 삭제 권한은 파일 자체가 아니라 **디렉토리의 쓰기(w) 권한**으로 결정됨 → sticky bit는 이 기본 규칙 위에 "본인/root만 삭제 가능"이라는 추가 제약을 건 것
- `ls -l`에서 마지막 자리가 `t`(소문자, other 실행권한 있음)와 `T`(대문자, 실행권한 없음) 차이 구분
- root는 sticky bit 예외 대상 → 실제 재현으로 확인 완료

## 보충 2. 하드링크 / 소프트링크

### Question
(덤프 문항 아님 — Day 4 보충 학습, inode 개념 및 링크 동작 검증)

### My Solution
```bash
echo "hello" > original.txt
ln original.txt hardlink.txt
ln -s original.txt softlink.txt
```

### Verification
```bash
$ ls -li original.txt hardlink.txt softlink.txt
17248997 -rw-r--r--. 2 root root  6 Jul  6 03:23 hardlink.txt
17248997 -rw-r--r--. 2 root root  6 Jul  6 03:23 original.txt
17248998 lrwxrwxrwx. 1 root root 12 Jul  6 03:23 softlink.txt -> original.txt

$ rm original.txt
$ cat hardlink.txt
hello                          # 정상 출력 (inode 살아있음)
$ cat softlink.txt
cat: softlink.txt: No such file or directory   # 깨진 링크
```

### 내가 헷갈렸던 것 / 배운 것
- 하드링크는 **같은 inode를 가리키는 이름표 하나 더**, 원본 삭제해도 살아남음 (링크 카운트가 0이 될 때만 진짜 삭제)
- 소프트링크는 **경로 문자열을 담은 별도 파일**, 원본 삭제 시 깨진 링크(broken link)가 됨
- 하드링크는 디렉토리 불가 + 같은 파일시스템 내에서만 가능 (`Invalid cross-device link` 에러), 소프트링크는 이 제약 없음

## 보충 3. Default ACL 
### Question
Allow the `finance` group to have Read and Execute (r-x) access to the `/groups/accounting` directory **and any new files created within it**.

### My Solution (예정)
```bash
groupadd finance
mkdir -p /groups/accounting
setfacl -R -m g:finance:r-x /groups/accounting
setfacl -d -m g:finance:r-x /groups/accounting
```

### Verification 
```bash
getfacl /groups/accounting
touch /groups/accounting/newfile.txt
getfacl /groups/accounting/newfile.txt   # default ACL 자동 상속 확인 필요
```

### 내가 헷갈렸던 것 / 배운 것
- 일반 ACL(`-m`)은 **지금 존재하는 파일에만** 적용, Default ACL(`-d`)은 **앞으로 생길 파일에 자동 상속되는 템플릿**
- 둘은 따로 관리됨 — 하나만 걸면 절반만 적용된 것 (기존 파일만 되거나, 새 파일만 되거나)
- `-d`는 디렉토리에만 의미 있음 (미래의 자식 항목을 위한 규칙이므로)

## 보충 4. 셸 스크립트 기초 
### Question
(덤프 문항 아님 — Day 4 보충 학습, 조건문/반복문/`$1` 기초)

### My Solution (예정)
```bash
#!/bin/bash
if id "$1" &>/dev/null; then
    echo "$1 사용자가 존재합니다."
else
    echo "$1 사용자가 존재하지 않습니다."
fi
```

### Verification 
```bash
chmod +x check_user.sh
./check_user.sh jacques
./check_user.sh nobody_user
```

### 내가 헷갈렸던 것 / 배운 것
- `[ ]`는 사실 `test` 명령어 — `[`가 독립된 명령어라서 안쪽에 공백 필수
- `chmod +x` 없으면 `./script.sh` 실행 불가 (`bash script.sh`로는 권한 무관하게 실행 가능)
- 변수는 항상 `"$1"`처럼 큰따옴표로 감싸는 습관 (빈 값/공백 포함 값일 때 문법 에러 방지)
